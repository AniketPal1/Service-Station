HOW THE DASHBOARD NOW WORKS
================================================================================

1. USER LOADS DASHBOARD.HTML:
   → dashboard-script.js loads
   → Page initialization runs
   → setActiveNavLink() highlights dashboard in nav
   → loadUserProfile() checks if user is logged in
   → displayBookingHistory() loads user's bookings
   → Events listeners attach to all buttons

2. IF USER IS LOGGED IN:
   → loadUserProfile() fetches user from LocalStorage
   → Profile card displays with user info
   → "View Profile" and "Logout" buttons show
   → displayBookingHistory() shows all user's bookings
   → Each booking shows as a card with details
   → Cancel/Reschedule buttons available

3. IF USER IS NOT LOGGED IN:
   → loadUserProfile() shows sign-in prompt
   → "Browse Services" button links to services.html
   → displayBookingHistory() shows sign-in prompt
   → User is prompted to sign in

4. USER CLICKS "VIEW PROFILE":
   → viewUserProfile() opens modal
   → Shows full user details
   → Displays edit profile button (placeholder)
   → User can close or edit

5. USER CLICKS "LOGOUT":
   → logout() removes currentUser from LocalStorage
   → Shows confirmation alert
   → Redirects to index.html
   → Dashboard shows sign-in prompt next visit

6. USER CLICKS "CANCEL" ON BOOKING:
   → cancelBooking() removes booking from array
   → Updates LocalStorage
   → displayBookingHistory() refreshes display
   → Booking disappears from list
